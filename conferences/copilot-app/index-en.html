The unlock: run agents in parallel

~20
PRs merged per day on boot-ui — impossible to type by hand
  • Each agent works in its own Git worktree — isolated, no collisions
  • The bottleneck moves from typing to reviewing
  • One person produces a small team's output

🏁 Hackathon move

  • Break your idea into independent tasks
  • Dispatch 3–5 agents at once
  • Review, then merge the winners
One developer + many parallel agents = a whole team's throughput, from a single laptop.

Practice 1 — Write the rules down

📜 AGENTS.md = a standing contract

  • The first file every agent reads
  • Durable rules & goals — not a throwaway plan
  • What the project is · its north-star · hard rules · how to build & test

🧱 Layer your context

  • AGENTS.md — the rules
  • copilot-instructions.md — the toolchain
  • Spec / plan / feature docs — the detail
"These are durable rules that should govern every future change." — from AGENTS.md
An agent is only as good as the context you give it. Write it once — reuse it forever.

Practice 2 — Give it the harness

The harness around the model matters more than the model itself.

⚡ Exact commands in the instructions

  • Build · run · single test · fast inner loop — spelled out
  • The agent iterates in seconds, not minutes

🧭 Real code intelligence

  • Wire a language server (LSP)
  • The agent navigates code precisely instead of guessing with text search
A great harness lets the agent run your tests and fix its own mistakes — the self-correcting loop.

Practice 3 — Guardrails let you say "yes" fast

✅ Automated gates

  • CI build
  • Tests / e2e
  • Format & lint
  • Security scan

📐 Encode your opinions

  • Turn "never do X" into a machine-checkable rule the build enforces

🔐 Safety rules

  • Never expose secrets / .env
  • Confirm destructive actions
Strong guardrails = you can accept high agent throughput without fear. CI is your co-reviewer.

Practice 4 — Small PRs, right model

🧩 Keep PRs small

  • Squash-merge focused changes
  • Review is now your main job
  • Small diffs = fast, safe review

🧠 Match model to task

  • Strong reasoning model for the bulk
  • 2–3 top models cross-checking the hard parts
  • Cheap / Auto for trivial work
You're the manager: assign the right agent to the right task — and review what comes back.

  • 1 · Write an AGENTS.md — your rules & goals
  • 2 · Wire the harness — exact build/test commands + code intelligence

  • 5 · Capture wins — turn them into a reusable skill
